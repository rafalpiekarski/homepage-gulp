GIT

Git to narzędzie do kontroli wersji.

Kroki do postępowania przy obsłudze GITa:

  1. Instalacja GITa ze strony www.git-scm.com
  2. Dodanie konta przy użyciu konsoli (jednorazowo): git config --global user.name "Imię Nazwisko"
  3. Dodanie konta-email (jednorazowo): git config --global user.email przykladowy_email@wp.pl
  4. Sprawdzenie stanu: git config --get user.name. Polecenie sprawdzi użytkownika. Analogicznie można zrobić dla adresu e-mail.
  5. Założenie repozytorium w katalogu, w którym mam pliki: git init
  6. Sprawdzenie statusu: git status
  7. Git pokaże, że pliki są nieśledzone: untracked files. Aby polecić GITowi śledzenie plików wpisuję: git add .. Kropka na końcu (koniecznie po spacji) pozwoli na dodanie wszystkich plików znajdujących się w katalogu. Mogę również dodawać pojedyncze pliki.
  8. Robienie commita: git commit -m "opis commita po angielsku".
  9. Nie commitujemy node_modules! - Każdy może sobie zrobić npm install i ściągnąć wymagane paczki
  10. .gitignore W folderze tworzę plik .gitignore, a w środku tworzę zapis: node_modules Dzięki temu git będzie wiedział, że ma ignorować plik node_modules
  11. git clone https://github.com/rafalpiekarski/homepage Pobieranie
  12. gitexplorer.com Łatwe narzędzie do tego, by uzyskać odpowiedź, jak coś zrobić w gicie.

Branchowanie

  1. Domyślna gałąź to master. Paczka gulpa sama stworzyła brancha na githuba i trzyma w nim zawartość katalogu dist. Ale branch gh-pages jest tylko na githubie. Nie ma go nawet lokalnie - jest usuwany od razu po wypchnięciu zmian na githuba.
  2. git branch Polecenie pozwala podejrzeć, jakie gałęzie są w projekcie.
  3. git branch nazwaBrancha Polecenie pozwala stworzyć gałąź o danej nazwie.
  4. git checkout nazwaBrancha przejście na gałąź o danej nazwie
  5. Mogę teraz tworzyć zmiany, commitować je. Podgląd w narzędizu gitk pokaże jak wygląda sytuacja na danej gałęzi - nowy commit jest tylko na nowej gałęzi.
  6. git branch master przechodzę na główną gałąź - mogę przechodzić po gałęziach.
  7. git push Jeśli będę chciał zapushować zmiany, to wyskoczy mi komunikat - to push the current branch and set the remote as upstream, use ... Powinienem użyć wtedy wyświetlonego polecenia: git push --set-upstream origin nazwaBranchaNieMaster
  8. git merge nazwaBranchaNieMaster Jeśli chcę zmiany z nowej gałęzi włączyć do głównej gałęzi master, muszę zmiany zmergować. Będąc na gałęzi master, wpiszę podane tu polecenie. Nasz master dogoni wtedy poboczny branch. Master ma ściągnięte zmiany z brancha bocznego.
  9. git pull Zrobiłem cos na githubie - na przykład przerobiłem plik redme. Mam teraz nieaktualną wersję u siebie. Robię polecenie git pull i ściągam dzięki temu zawartość z githuba do siebie na repozytorium.

SASS

  • SASS jest narzędziem, ma swoją składnię i ma swoją składnię SASSową. Będziemy korzystać ze składni SCSS.
  • .SCSS jest dla programisty, kod przechodzi przez SASS i trafia jako .CSS do przeglądarki.
  • www.sassmeister.com Strona do przetwarzania kodu sassowego na .css
  • $favouriteColor: #ccc; Zadeklarowałem zmienną,w której przetrzymuję ulubiony kolor.
  • .main { color: $favouriteColor;} Zastosowałem zmienną do właściwości color.
  • $desktop: 1024px; @media (min-width: $desktop;) Zastosowałem deklarację zmiennej do tego, by użyć jej w zapytaniu @media
  • @mixin borderBackground { border-radius: 10px; background: hotpink;} .main {@include borderBackground;} W mixinie zawarte ileś tam własności. Dopisują się one w pliku .css
  • @mixin borderBackgroundParameter($radius, $background) {border-radius: $radius; background: $background;} Przydatne. Tworzę 'funkcję', któe przyjmują parametry. Stosowane w generatorach bądź przy tworzeniu efektów.
  • .font-style {font-size: 13px; color: green;}
    .main {color: $favouriteColor; @include borderBackgroundParameter(13px, blue);
    @extend .font-style;
     Wszystkie wspólne style z klas .main oraz .font-style zostały wrzucone na górę arkusza .css. Stosując @mixin własności dopiszą się do klas, a tutaj wspólne zostały wyrzucone na górę. Lepiej korzystać z mixinów i include.
  • Nie modyfikujemy plików .CSS jeśli mamy pliki .SCSS

Node.js oraz NPM

  3. node -v Aby sprawdzić, czy node się zainstalował.

  5. node script.js Uruchomienie skryptu o nazwie script.js.
  1. NPM Wyszukiwarka pakietów, repozytorium pakietów, sklep
  2. Paczki - dodajemy sporo gotowych bibliotek bądź narzędzi do projektu. Jet to uniwersalne źródło ze wszstkimi narzędziami, z dostępdem do aktualizacji.

  4. Npm będzie teraz zadawać pytania: nazwa paczki. W nawiasie package name: (cwiczenia) podana jest wartość domyślna. Jeśli pasuje, to można zatwierdzać.

  11. Można nawet skasować folder node_modules - informacje o tym, jakie paczki są potrzebne jest we właściwości dependencies. Wystarczy wpisać npm install

Pobierając od kogoś projekt wiemy, jakie wersje których paczek są w projekcie - nie wyjdzie sytuacja, gdy mamy np. nowszą paczkę, która jest niekompatybilna z wersją zastosowaną w projekcie.
  13. npm audit Polecenie sprawdza stan aktualizacji i błędów w pakietach npm. Drugim poleceniem powinno byc npm audit fix Aby npm spróbował naprawić zauważone błędy.

PWA - Progressive Web Apps

  • Aplikacje natywne pisane w technologiach HTML, CSS, JS - bez konieczności nauki nowych technologii (powiadomienia, aparat itp). Działają offline, łatwa aktualizacja i dystrybucja.
  • Technologia w trakcie rozwoju, ale bardzo przyszłościowa
  • Umożliwia stosowanie jednego kodu
  • Potrzebne są dwa pliki do uruchomienia PWA: serviceworker.js oraz manifest.json (lub inna nazwa, nie ma jednego standardu na to).

  • W pliku manifest.json istotne jest scope oraz start_url, które w przypadku adresu np.: https://rafalpiekarski.github.io/homepage-gulp/ powinno mieć formę: /homepage-gulp/
  • wchodzimy w chrome w zakładkę application. Mamy zakładkę Manifest oraz zakładkę Service Workers.
  • Wszedłem na adres strony na githubie: w simple-editor. Otwierając ten adres na telefonie, android zapyta sie, czy dodać aplikację do ekranu głównego. Mam już wtedy stronę w formie aplikacji na telefonie. Odpalając npm run deploy ona też się powinna zaktualizować (?).

Przejścia i animacje

Transition

  • Przejście ustawia się na elemencie, który ma się zmienić, a nie na wyzwalaczu (triggerze) - mówimy elementowi "elemencie, jeśli będziesz się zmieniał, to rób to np. przez sekundę"
  • Mechanizmem wyzwalającym w CSS jest zazwyczaj :hover. w JS może to być już np. kliknięcie. Najczęściej spotykany sposób na zastosowanie jakiegoś efektu to dodawanie / odejmowanie klasy w zależności od potrzeb (np. button vs button.show).
  • co warto animować: kolory, opacity, pozycję, model pudełkowy (margin, padding, border), transformacje (rotate, scale).
  • Do czego nie warto stosować transition: display, float, position, rodzaj czcionki, width, height - są to właściwości, których przeliczanie bardzo mocno obciąża przeglądarkę.
  • transition all .3s ease-in-out Niepolecane jest, aby przejścia stosować dla wszystkich właściwości, gdyż bardzo to obciąża zasoby.
  • transition: (none) (duration) (timing function) (delay)Aby użyć transition potrzeba: wartośći początkowej + wartości końcowej + określenia samego przejścia + zdefiniowanie triggera (wyzwalacza)
  • Przykład zastosowania transition dla kilku właściwośći: transition: color 1s ease, font-size 1s 2s linear, top 1s ease-in 3s;.

Transform

  • translate przemieszcza dany element. Odnosi się do elementu, a nie do rodzica!. Właśnie dzięki temu może być użyty bardzo dobrze do wysrodkowania elementu.
  • transform można zrobić np. na tekście.
  • transform: translate(10rem, -5%); przesunięcie o zadaną wartość w osi X oraz Y
  • transform: scale(); powiększa bądź pomniejsza element. scale(-1) umożliwia obrócenie lustrzane elementu.
  • transform: translateX(200px); transform: translateY(100px); Wykona się tylko translateY, ponieważ translateX zostanie przez translateY nadpisane. Wynika to z tego, że są to elementy jednej zbiorczej właściwości translate.
  • Jeśli napiszę div {transform: rotate(60deg) translateX(200px);} to nie mogę napisać div:hover {transform: rotate(30deg);. Ponieważ właściwość translateX nie wystąpiła, to będzie miała domyślną wartość, tutaj wynosi ona (0), więc zostanie nadpisana z wartością (0).
  • transform: translate(100px) oznacza to samo, co translate(100px, 100px).
  • scaleY(-100px); transform: scaleX(-100px); Nie można tak napisać, ponieważ pierwsze scaleY zostanie nadpisane przez to, co jest w scaleX. (do sprawdzenia)
  • Dodawanie więcej niż jednej transformacji translate - jedna po drugiej jest bez przecinka.
  • Bardzo ważna jest kolejność tranformacji: tranlate() transform() da inne wyniki, niż transform() translate()

Animacje

  • animation-play-state: paused; Animacja zatrzymuje się / zamraża.
  • both oznacza, że animacja weźmie punkt 0% oraz 100% spoza animacji jako punkty startowe. Mam pozycję np. margin-left: 20px;, a w animacji przy 0% jest margin-left: 200px. Animacja zaczynając się od razu przeskoczy na 200px. Wartość BOTH zmieni 0% oraz 100% na wartości tuż przed/po animacji
  • steps(4); animacja wykona się nie płynnie, tylko w 4 krokach
  • animation-fill-mode Jest ważne gdy animacja nie uruchamia się od razu, a np. po kliku lub animacja nie jest nieskończona, ma jakieś zakończenie. W wartości 'none' to co jest poza animacją decyduje o stanie przed i po animacji. Jeśli nie ma opóźnienia animacji, to ona i tak od razu się zacznie, więc nie trzeba się przejmować animation-fill-mode. Mając animation-fill-mode: both; nie interesuje nas co jest przed animacją, tylko własności 0% i 100% animacji. Jest jeszcze forwards i backwards. Przydatne
  • Dwie wymagane wartości: animation-name, animation-duration. Domyślnie duration =0 i name = none
  • Letter spacing można animować - ciekawostka.

BEM - Block Element Modifier

Opis poszczególnych składowych metodologii BEM:

  • Blok - element, którego nie dzielę na mniejsze częśći, a elementy wewnątrz niego są ze sobą ściśle powiazane. Przykładem możę być stopka, ale też i przycisk.
  • Element - znaczniki powiązane ściśle z blokiem. Przykładem może być ikona przycisku lub tekst przycisku.
  • Modyfikator - w selektorze modyfikatora zapisujemy tylko wartości, które on modyfikuje.
  • block__element--js Służy do umieszczdenia klasy, której będziemy używać w JS.

Ważna jest kolejność zapisu. Najpierw zapisujemy selektor dla bloku, dopiero później tworzymy selektor modyfikatora.

Kiedy wybrać blok, a kiedy element? Może się zdarzyć, że wpadniemy w zakłopotanie, czy dany dany fragment nazwać blokiem, czy elementem. W takich momentach powinniśmy chwilę się zastanowić i wybrać jedną możliwość. Jeżeli kod będzie używany w wielu miejscach na stronie, a jego wygląd ma być niezależny od miejsca występowania — powinniśmy stworzyć blok. Jeżeli kod będzie nieodłączną częścią rodzica i będzie od niego zależne — powinniśmy stworzyć element.

Modyfikator musi zawsze iść w parze z oryginalnym elementem. Tak jest dobrze: .blok__element .blok__element--modyfikator . Tak nie jest dobrze: .blok__element--modyfikator

Nie zagnieżdżamy więcej niż raz. Strukturra blok - element - modyfikator. Blok nie musi mieć elementów (blok to może być 'articles, a w nim moga być zawarte bloki 'article').

Dodawanie modyfikatora JSowego do istniejącego już modyfikatora (nie do elementu, a właśnie modyfikatora) poprzez pojedynczą kreskę: button class="button button--add button--add-s"

FLexbox

  • W bardzo prostej stronie można nawet użyć tylko flexboxa zamiast gita.
  • Polecenie display: flex nadaję rodzicowi elementów, które chcę układać flexem. Od razu po nadaniu tej właściwości zmienia się układ.
  • Flex-direction: row, column, row-reverse, column-reverse .Domyślna wartość to ROW - elementy układają się pionowo jeden obok drugiego od lewej do prawej. Zmieniając na COLUMN układ będzie pionowy - elementy będą się układać od góry do dołu. Czasem aby zadziałało dobrze COLUMN, to dobrze mieć wysokość elementów
  • Flex-wrap: nowrap, wrap, wrap-reverse. Domyślnie NO-WRAP. Zawijanie wiersza bądź kolumny. Domyślnie elementy ścieśnią się aby nie wychodzić za obszar rodzica. Mając WRAPA elementy się zawiną do kolejnego wiersza bądź kolumny. WRAP-REVERSE zrobi tak, że zawijanie będzie np. od dołu szło - zamienia wiersze, aby szły od dołu.
  • Flex-flow: (flex-direction) (flex-wrap) Zbiorcza właściwość dla DIRECTION i WRAP.
  • justify-content: (flex-start) (flex-end) (center) (space-between) (space-around) Ustawia elementy wzdłuż osi FLEX-DIRECTION. Jeśli mam elementy ustawione w wierszach, to własność FLEX-END sprawi, że wiersze przykleją się do prawej krawędzi rodzica. Mając ustawione CENTER - kazdy wiersz wyśrodkuje się względem rodzica.
  • align-content: (stretch) (flex-start) (flex-end) (center) (space-between) (space-around). Domyślną wartością jest STRETCH. Np. dla elementów w 2 wierszach, każdy z nich zajmuje po połowie wysokości - bo jest rozciągnięty. Ustawia wiersz bądź kolumnę w osi przeciwnie do flex-direction. Jeśli ustawię np. (CENTER) to nie mając podanej wysokości, elementy się skurczą.
  • align-items: (stretch) (flex-start) (flex-end) (center) (baseline) Ustawia elementy wzdłuż osi przeciwnej do FLEX-DIRECTION ale w obrębie pojedynczego wiersza. Jeśli jeden element jest np. wyższy od pozostałych, to tutaj wypozycjonujemy elementy w wierszu.
  • align-self: (center) Ustawi pojedynczy element np. centralnie w wierszu z innymi elementami - tyczy się tylko jednego elementu. jest to właściwość ALIGN-ITEMS dla pojedynczego elementu.
  • order Można wskazać elementom kolejność.
  • flex-grow Jak bardzo element rośnie. Mając flex-grow:1; nie będzie wolnego miejsca, wszystkie będą miały tę samą szerokość. Jeśli elementowi nadam flex-grow:2; to będzie on 2x szerszy od pozostałych elementów z flex-frow:1;
  • flex-shink Ustawione na "0" sprawia, że elementy kurczą się maks. do swoich rozmiarów. Dobrze działa, gdy ustawimy wszystkim elementom jakąś wartość, a niektórym np. 2x większa wartość
  • flex-basis Rozmiar bazowy. Określenie minimalnej szerokości lub wysokości elementu.
  • flex: (flex-grow) (flex-shrink) (flex-basis) Własność zbiorcza.

Przykład: console.log(a); W tej sytuacji konsola wywali komunikat, że a jest niezdefiniowane ('Uncaught ReferenceError: a is not defined'). console.log(a); var a=7; W tej sytuacji konsola nie wywali błędu, że a jest niezdefiniowane, ale pokaże, że jest wartość 'undefined'. Najpierw jest odwołanie się do 'a', a dopiero później jest sama deklaracja. Pomimo tego, przeglądarka wie, że wartość 'a' została zadeklarowana - najpierw nastąpił hoisting, czyli odczytanie zmiennych - tak, jakby deklaracja zmiennej (bez jej wartości) były wywindowane na górę kodu.

Przechowywanie danych

  • Cookies - trochę przestarzałe. Obecnie stosuje się local stories.
  • document.cookie Dobranie się do ciastek z dokumentu.
  • document.cookie = "nazwa = wartość"; Dodawanie cookie, które ma nazwę i daną wartość.
  • document.cookie = "nazwa = wartośćKolejna"; Dodawanie kolejnego cookie, które ma nazwę i daną wartość. Nie nadpisze się kolejne, tylko doda.
  • Ciastka nie są tak istotne dla początkującego frontendowca. Są też trochę archaiczne. Ważniejsze są web storage: local storage oraz session storage.
  • localStorage.setItem('nazwa', 'wartość'); Mtoda wstawi do Local Storage klucz o nazwie 'nazwa' mający wartość 'wartość'.
  • localStorage.setItem('person', person); Chcemy do Local Storage wstawić obiekt o nazwie person. Ustawiłem nazwę na person a w wartości jest obiekt person (stworzony wcześniej, tutaj tylko przykładowo). Wchodząc teraz w tę local storage w wartości będzie pisało [object Object] - po prostu obiektu nie można wrzucić w ten sposób do local storage.
  • JSON.stringify(person) Jeżeli chcemy zapakować obiekt do Local Storage, to musimy użyć metody JSON.stringify - zamienimy go w string JSONowy i on trafi w takiej formie do Local Storage.
  • console.log(JSON.stringify(person)); Używam console.log aby sobie sprawdzić, jak taki obiekt teraz wygląda. Jak widać w console log, stworzył się jednolinijkowy string z danymi. Można go wrzucić do Local Storage właśnie w tej formie - JSONowego stringa - mogę użyć localStorage.setItem aby wstawić do Local Storage taki ciąg.
  • console.log(localStorage.getItem('name')); Pobieram wartość, jaka jest w kluczu o nazwie 'name' i wypisuje ją w konsoli - dostaję wartość z klucza.
  • console.log(localStorage.getItem('person')); Problem: Pobiore dane w formie stringa JSONowego, bo takie były w Local Storage. Będę więc musiał później zamienić ten string na obiekt.
  • const personStringify = localStorage.getItem('person'); const newPerson = JSON.parse(personStringify); Najpierw zapisuję do zmiennej pobrany string zawierający dane z obiektu person. Później mogę go zamienić na obiekt
  • localStorage.removeItem('nazwa'); usuwam dany klucz z Local Storage.

Praca z API / JSON

  • Pobieranie danych z zewnątrz w taki sposób, aby przeglądarka klienta sama sobie pobrała te dane a twórca nie musiał o tym pamiętać
  • API pozwala na wykonywanie operacji (pobieranie, dodawanie, usuwanie, modyfikacja) w aplikacji bez konieczności znania jej budowy
  • JS udostępnia szereg różnych API. Wiele gotowych produktów udostępnia swoje API (Google, slack, github, ...). Local Storage jest też takim rodzajem API - piszemy localStorage.getItem - pobieramy.
  • Idziemy do sklepu i chcemy coś kupić - sprzedawca nam ją podaje a my nie musimy wiedzieć, gdzie ona leży i jak wygląda.
  • REST - Representational State Transfer - przykład architektury API.
  • GraphQl - zaczyna być popularny, powoli wypiera RESTa
  • Łączenie się z przykładowym API: API Githuba. Pomoce nas stronie: developer.github.com Wchodzimy w sekcję Dosc / REST API V3.
  • Pobieramy i instalujemy program postman: getpostman.com Pobieramy aplikację. Nie trzeba się logować.
  • Metodą post odpytajmy w prorgamie Postman następujący adres: dev.cotenfrontend.pl
  • Możemy wysłać zapytanie metodą GET z parametrami: np. key=name + value=Rafal lub przykładowo drugie: key=age + value=32.
  • Metodą POST nie uda się tego zrobić w taki sposób. Ttzeba wybrać Body / x-www-form-urlencoded
  • Na stronie Githuba we wspomnianej uprzednio sekcji REST API V3, w tytule sekcji 'Current version widnieje adres: https://api.github.com. Jest to adres do kominikacji z API.
  • Wchodzimy w zakładkę 'repositories' i np. w sekcję 'List user repositories'. Widnieje tam polecenie: 'GET /users/:username/repos'.
  • Docelowym poleceniem, jaki epwiszę w metodzie GET w programie POSTMAN będize więc: https://api.github.com/users/rafalpiekarski/repos
  • Parametry, jakie można wpisać szukając repozytoriów, to (dla strony Github) 'type', 'sort', 'direction'. Jesli chciałbym posortować np. według daty tworzenia, to zrobiłbym to następująco: https://api.github.com/users/rafalpiekarski/repos?sort=created. Postman pomaga w tym.
  • Często pracując z jakimś projektem dostajemy listę tego, co można i gdzie odpytać.
